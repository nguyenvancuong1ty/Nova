# PHASE 1 PLAN — Novel-to-Video Script Production Workflow

## 1. Phase 1 Objective

Build a private local workflow app that allows one creator to input core novel information, enter the desired number of chapters, click Start, and generate a complete production package for a long-form WebNovel / Light Novel series.

Phase 1 does **not** generate real images or real videos.

Phase 1 generates:

* Story Bible
* World Bible
* Character Bible
* Visual Reference Descriptions
* Arc Plan
* Chapter Plan
* Chapter Drafts
* Scene Breakdowns
* Continuity Reports
* Revised Final Chapters
* Storyboards
* Image Prompts
* Video Prompts
* Voiceover Scripts
* Subtitle Drafts
* Export Package

This phase should behave like a **Novel-to-Video Script Production Engine**, not a final video rendering engine.

---

## 2. Product Direction

This project is a private local workflow app for one creator.

The primary user flow is:

1. User opens a local web UI.
2. User fills in a novel production form.
3. User enters total chapter count.
4. User clicks **Start**.
5. The system runs the entire production workflow end-to-end.
6. The system writes all generated files into an output project folder.
7. The user reviews the generated text, storyboard, prompts, and production assets.

The user should not manually run separate commands for every chapter.

CLI may exist only as a secondary developer interface.

The main interface is a local web UI with a Start button.

## 7. Backend API Requirements

Create backend endpoints.

### POST /api/production/start

Starts a new production workflow.

Request body:

```json
{
  "project": {},
  "story": {},
  "world": {},
  "characters": [],
  "video": {},
  "chapterConfig": {}
}
```

Response:

```json
{
  "runId": "run_...",
  "projectSlug": "shadow-crown",
  "status": "started"
}
```

### GET /api/production/status/:runId

Returns current workflow progress.

Response:

```json
{
  "runId": "run_...",
  "status": "running",
  "currentStep": "generate_chapter_draft",
  "currentChapter": 3,
  "totalChapters": 10,
  "progressPercent": 32,
  "logs": []
}
```

### GET /api/production/result/:runId

Returns final output information.

Response:

```json
{
  "runId": "run_...",
  "status": "completed",
  "outputPath": "outputs/shadow-crown",
  "generatedFiles": []
}
```

### POST /api/production/cancel/:runId

Cancels a running workflow.

For Phase 1, cancellation can be simple:

* Mark run as cancelled.
* Stop before starting the next chapter.

## 9. Data Schemas

Use Zod for all important data.

### ProductionInput

```ts
export const ProductionInputSchema = z.object({
  project: ProjectConfigSchema,
  story: StoryInputSchema,
  world: WorldInputSchema,
  characters: z.array(CharacterInputSchema).min(1),
  video: VideoConfigSchema,
  chapterConfig: ChapterConfigSchema
});
```

### ProjectConfig

```ts
export const ProjectConfigSchema = z.object({
  title: z.string().min(1),
  creatorName: z.string().optional(),
  language: z.string().default("Vietnamese"),
  format: z.enum(["webnovel", "light-novel", "serial-fiction"]),
  genres: z.array(z.string()).min(1),
  tone: z.string().min(1),
  targetAudience: z.string().optional()
});
```

### ChapterConfig

```ts
export const ChapterConfigSchema = z.object({
  totalChapters: z.number().int().min(1).max(300),
  targetWordsPerChapter: z.object({
    min: z.number().int().min(500),
    max: z.number().int().min(500)
  }),
  scenesPerChapter: z.object({
    min: z.number().int().min(3),
    max: z.number().int().min(3)
  })
});
```

### CharacterInput

```ts
export const CharacterInputSchema = z.object({
  id: z.string().min(1),
  name: z.string().min(1),
  role: z.enum([
    "protagonist",
    "antagonist",
    "supporting",
    "mentor",
    "love_interest",
    "rival",
    "other"
  ]),
  age: z.number().int().optional(),
  visualDescription: z.string().min(1),
  personality: z.array(z.string()).default([]),
  motivation: z.string().optional(),
  fear: z.string().optional(),
  secret: z.string().optional(),
  voice: z.string().optional(),
  relationships: z.array(
    z.object({
      characterId: z.string(),
      relation: z.string(),
      notes: z.string().optional()
    })
  ).default([])
});
```

### Scene

```ts
export const SceneSchema = z.object({
  id: z.string(),
  chapterNumber: z.number().int(),
  sceneNumber: z.number().int(),
  title: z.string(),
  location: z.string(),
  characters: z.array(z.string()),
  purpose: z.string(),
  summary: z.string(),
  visualFocus: z.string(),
  dialogueFocus: z.string().optional(),
  emotionalBeat: z.string(),
  videoPotential: z.enum(["low", "medium", "high"])
});
```

### ImagePrompt

```ts
export const ImagePromptSchema = z.object({
  id: z.string(),
  chapterNumber: z.number().int(),
  sceneNumber: z.number().int(),
  prompt: z.string(),
  negativePrompt: z.string().optional(),
  aspectRatio: z.enum(["16:9", "9:16", "1:1"]),
  style: z.string(),
  characterReferences: z.array(z.string()),
  characterConsistencyNotes: z.array(z.string())
});
```

### VideoPrompt

```ts
export const VideoPromptSchema = z.object({
  id: z.string(),
  chapterNumber: z.number().int(),
  sceneNumber: z.number().int(),
  prompt: z.string(),
  cameraMovement: z.string(),
  durationSeconds: z.number().int(),
  motionNotes: z.string(),
  soundDesignNotes: z.string().optional()
});
```

### RunState

```ts
export const RunStateSchema = z.object({
  runId: z.string(),
  projectSlug: z.string(),
  status: z.enum(["idle", "running", "completed", "failed", "cancelled"]),
  currentStep: z.string().optional(),
  currentChapter: z.number().int().optional(),
  totalChapters: z.number().int(),
  progressPercent: z.number(),
  startedAt: z.string(),
  finishedAt: z.string().optional(),
  outputPath: z.string(),
  error: z.string().optional(),
  logs: z.array(z.string()).default([])
});
```

## 10. Core Workflow

Implement main workflow:

```ts
runNovelProduction(input: ProductionInput): Promise<ProductionResult>
```

Workflow steps:

```text
1. validateInput
2. createProjectFolder
3. saveOriginalInput
4. generateStoryBible
5. generateWorldBible
6. generateCharacterBible
7. generateVisualReferenceBible
8. generateArcPlan
9. generateChapterPlan
10. for each chapter:
    10.1 generateChapterOutline
    10.2 generateChapterDraft
    10.3 splitChapterIntoScenes
    10.4 runContinuityCheck
    10.5 reviseChapter
    10.6 generateStoryboard
    10.7 generateImagePrompts
    10.8 generateVideoPrompts
    10.9 generateVoiceover
    10.10 generateSubtitles
    10.11 updateTimeline
11. exportFullNovel
12. exportProductionAssets
13. writeFinalSummary
14. markRunCompleted
```

All generation in Phase 1 must be deterministic placeholder generation.

No AI calls.

---

## 11. Step Specifications

### 11.1 validateInput

Validate the input using Zod.

Fail if:

* title is missing
* total chapters is invalid
* no protagonist exists
* target word count min > max
* scenes per chapter min > max
* visual style is missing

Output:

* validated input object

---

### 11.2 createProjectFolder

Create a slug from the project title.

Example:

```text
The Shadow Crown
```

becomes:

```text
the-shadow-crown
```

If folder exists, append timestamp:

```text
the-shadow-crown-2026-07-07-2230
```

Create all required folders.

---

### 11.3 saveOriginalInput

Save:

```text
input/production-input.json
```

This file is important because the whole project can be regenerated from it later.

### 11.7 generateVisualReferenceBible

Create:

```text
knowledge-base/visual-reference-bible.md
```

This file is critical for future image/video consistency.

For each character, include:

* Stable visual identity
* Face description
* Hair
* Eyes
* Body type
* Clothing style
* Signature colors
* Accessories
* Expressions
* What must not change
* Prompt fragment

Example:

```md
## Character: Arin

Reference ID: character:arin

Stable Visual Identity:
Arin is a young male protagonist with silver-gray eyes, messy black hair, a lean frame, and a long dark travel coat.

Must Not Change:
- Eye color must remain silver-gray.
- Hair must remain black.
- Body type must remain lean.
- Signature coat must remain dark.

Prompt Fragment:
Use character reference character:arin. Maintain silver-gray eyes, messy black hair, lean frame, and long dark travel coat.
```

---

### 11.8 generateArcPlan

Create:

```text
planning/arc-plan.json
```

For Phase 1, generate arcs deterministically based on total chapter count.

Recommended logic:

* 1–10 chapters: 1 arc
* 11–30 chapters: 3 arcs
* 31–60 chapters: 5 arcs
* 61–100 chapters: 7 arcs
* 100+ chapters: 10 arcs

Each arc should include:

* id
* title
* startChapter
* endChapter
* premise
* mainConflict
* keyReveals
* emotionalProgression
* expectedEnding

---

### 11.9 generateChapterPlan

Create:

```text
planning/chapter-plan.json
```

Generate one entry per chapter.

Each chapter plan should include:

* chapterNumber
* title
* arcId
* mainGoal
* mainConflict
* requiredCharacters
* requiredLocations
* emotionalBeat
* mysteryProgress
* romanceProgress
* endingHook
* videoFocus

For Phase 1, placeholder generation is acceptable, but the output should look structurally realistic.

### 11.10 generateChapterOutline

* Emotional beat

### 11.11 generateChapterDraft

Create:

```text
chapters/chapter-0001/chapter-draft.md
```

For Phase 1:

* Do not attempt to generate a perfect long chapter.
* Generate deterministic placeholder prose.
* The draft should include scene headings.
* Each scene should contain enough content for testing downstream steps.

Structure:

```md
# Chapter 0001 — Title

## Scene 01 — Scene Title

[Placeholder prose...]

## Scene 02 — Scene Title

[Placeholder prose...]
```

---

### 11.12 splitChapterIntoScenes

Create:

```text
chapters/chapter-0001/scenes.json
```

Generate scenes based on chapter config.

Each scene should reference character IDs, not full descriptions.

Important:

```json
{
  "characters": ["protagonist"]
}
```

Do not duplicate full visual descriptions inside scenes.

Visual descriptions should come from the visual reference bible later.

---

### 11.13 runContinuityCheck

Create:

```text
chapters/chapter-0001/continuity-report.md
```

Check:

* Scene count is within configured range
* All scene character IDs exist
* All scene locations are not empty
* Every scene has a visual focus
* Every scene has an emotional beat
* Every scene has a purpose
* Chapter has an ending hook
* No unknown character references
* No missing required characters from chapter plan

Report format:

```md
# Continuity Report — Chapter 0001

## Status

PASS / WARN / FAIL

## Issues

## Character Consistency

## Timeline Consistency

## Worldbuilding Consistency

## Visual Consistency

## Video Adaptation Notes

## Recommended Fixes
```

---

For Phase 1:

* If no major issue: copy draft to final and add a revision note.
* If issues exist: add a "Revision Notes" section at the bottom.

### 11.16 generateImagePrompts

Create:

```text
chapters/chapter-0001/image-prompts.json
```

Prompt must include:

* Location
* Scene mood
* Visual focus
* Character reference IDs
* Style
* Aspect ratio
* Consistency notes

Important:

Do not invent new character appearances.

Use references:

```text
character:<character-id>
```

```json
  "chapterNumber": 1,
```

## 14. Error Handling

The app should fail clearly.

Examples:

### Invalid input

```text
Invalid production input:
- project.title is required
- chapterConfig.totalChapters must be at least 1
```

### Unknown character

```text
Continuity warning:
Scene 3 references unknown character: "shadow_king"
```

### File write error

```text
Failed to write file:
outputs/the-shadow-crown/chapters/chapter-0001/scenes.json
```

When an error happens:

* update run-state.json
* set status to failed
* write error to logs
* return clear API response
* show error in UI
