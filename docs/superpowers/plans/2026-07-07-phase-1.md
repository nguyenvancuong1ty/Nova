# Phase 1 Novel Production Engine Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a local React + Express app that accepts novel production input, runs a deterministic text-only workflow, tracks progress, and writes the complete Phase 1 output package to the filesystem.

**Architecture:** Use a single TypeScript workspace with Vite for the frontend and an Express server for the API. Keep generation logic in small workflow step modules, centralize filesystem and run-state behavior in services, and make the workflow testable without the UI by calling `runNovelProduction()` directly.

**Tech Stack:** Node.js, TypeScript, Vite, React, Express, Zod, `fs/promises`, Vitest, ESLint, Prettier

---

// src/services/runRegistry.ts
import type { RunState } from "../types";

export class RunRegistry {
  private readonly runs = new Map<string, RunState>();
  private readonly cancellations = new Set<string>();

  set(state: RunState) {
    this.runs.set(state.runId, state);
  }

  get(runId: string) {
    return this.runs.get(runId);
  }

  cancel(runId: string) {
    this.cancellations.add(runId);
  }

  isCancelled(runId: string) {
    return this.cancellations.has(runId);
  }
}
